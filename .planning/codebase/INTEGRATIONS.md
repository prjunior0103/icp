## API Routes Architecture

The ICP application exposes 26 API routes in `/app/api/` for managing the incentive compensation platform. All routes follow Next.js 16 API route conventions (GET, POST, PUT, DELETE handlers).

## Authentication Flow

### NextAuth.js Integration
- **Route**: `/api/auth/[...nextauth]` → `auth.ts` handler
- **Provider**: Credentials (username/password)
- **Flow**:
  1. Form submission at `/login` with credentials
  2. `authorize()` callback validates against `AUTH_USERNAME` and `AUTH_PASSWORD` from env
  3. On success: returns user object with `{ id, name, role: 'GUARDIAO' }`
  4. JWT callback enriches token with role claim
  5. Session callback injects role into `session.user.role`
  6. All subsequent requests use JWT strategy (no sessions)
- **Session Storage**: JWT in secure HTTP-only cookie
- **Role Propagation**: Token.role → session.user.role for authorization checks

## External Data Import/Export Routes

### 1. Indicadores (Indicators)
**Route**: `/api/import-indicadores`
- **GET**: Returns CSV template for bulk indicator upload
  - Headers: codigo, nome, tipo, polaridade, abrangencia, unidade, metaMinima, metaAlvo, metaMaxima, diretivo, analistaResp, origemDado
  - Example row provided
- **POST**: Bulk upsert indicators via CSV rows
  - Input: `{ cicloId: number, rows: Record<string, string>[] }`
  - Logic: Find or create Indicador by codigo (unique constraint)
  - Validations: VOLUME_FINANCEIRO|CUSTO_PRAZO|PROJETO_MARCO|QUALITATIVO types
  - Validations: MAIOR_MELHOR|MENOR_MELHOR polarity
  - Validations: CORPORATIVO|AREA|INDIVIDUAL scope
  - Output: `{ processed: number, updated: number, erros: { linha, motivo }[] }`

### 2. Colaboradores (Employees)
**Route**: `/api/import-colaboradores`
- **GET**: Returns CSV template with headers + example
  - Headers: matricula, nomeCompleto, cpf, email, salarioBase, dataAdmissao, empresaCodigo, cargoCodigo, centroCustoCodigo, gestorMatricula
- **POST**: Bulk upsert employees with auto-creation of linked entities
  - Input: `{ rows: Record<string, string>[] }`
  - Deduplication: By matricula (unique)
  - Auto-creates: Empresa, Cargo, CentroCusto if not found
  - Optional: cargoNome, nivelHierarquico, targetMultiploSalarial can be provided per row
  - Manager linking: gestorMatricula resolves to existing Colaborador.id
  - Normalization: Strips UTF-8 BOM, trims whitespace, handles comma/dot decimal swapping
  - Output: `{ processed: number, updated: number, erros: { linha, motivo }[] }`

### 3. Metas (Goals)
**Route**: `/api/import-metas`
- **GET**: Returns CSV template
  - Headers: indicadorCodigo, pesoNaCesta, metaMinima, metaAlvo, metaMaxima, parentMetaIndicadorCodigo, smart_e, smart_m, smart_a, smart_r, smart_t
- **POST**: Bulk create metas (cascading goals)
  - Input: `{ cicloId: number, rows: Record<string, string>[] }`
  - Logic: Maps indicadorCodigo → indicador.id
  - Parent linking: parentMetaIndicadorCodigo → resolves parent Meta via Indicador
  - SMART fields: JSON-serialized as `{ e, m, a, r, t }` if any provided
  - Output: `{ processed: number, erros: { linha, motivo }[] }`

### 4. Bulk Realizacoes Import
**Route**: `/api/bulk-import`
- **POST**: Import performance results (realizacoes) for a month/year
  - Input: `{ cicloId, mesReferencia, anoReferencia, rows: ImportRow[] }`
  - ImportRow shape: `{ matricula, metaCodigo, valorRealizado, observacao? }`
  - Calculation: Calls `calcularNota()` and `calcularPremio()` from business logic
  - Upsert: By unique constraint (metaId, colaboradorId, mesReferencia, anoReferencia)
  - Status: Auto-set to "SUBMETIDO" with timestamp
  - Output: `{ processed: number, erros: { matricula, motivo }[] }`

## Calculation Engine

### File: `/app/lib/calc.ts`
**Business Logic Functions** (pure, testable):
- **calcularNota(tipo, polaridade, valorRealizado, metaAlvo, metaMinima, metaMaxima)**: number
  - PROJETO_MARCO: Binary (100 if achieved, 0 otherwise)
  - MENOR_MELHOR: Inverse ratio (metaAlvo / valorRealizado × 100)
  - MAIOR_MELHOR: Direct ratio (valorRealizado / metaAlvo × 100)
  - Capping: Max score 120% unless PROJETO_MARCO
  - Floor: 0 if below minimum (MAIOR_MELHOR) or above maximum (MENOR_MELHOR)
- **calcularPremio(salarioBase, targetMultiploSalarial, nota, pesoNaCesta)**: number
  - Formula: salarioBase × targetMultiploSalarial × (nota / 100) × (pesoNaCesta / 100)
- **normalizarLinhaColaborador(raw)**: Strips BOM, trims whitespace
- **parseSalario(raw)**: Parses Brazilian format (8.000,50 → 8000.50)
