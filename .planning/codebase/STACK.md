## Dev vs Production Differences

### Development
- SQLite database: `file:./dev.db`
- Prisma client is cached in `globalForPrisma` to prevent connection pool exhaustion
- Default auth credentials: `admin` / `admin` (from .env)
- SourceMap generation: enabled for debugging

### Production
- **Database**: Turso cloud (LibSQL remote instance)
  - Requires `TURSO_DATABASE_URL` and `TURSO_AUTH_TOKEN` environment variables
  - Uses `PrismaLibSql` adapter for remote connection pooling
- **Auth Secret**: Must be set via `AUTH_SECRET` environment variable (currently unsafe default in .env)
- **Node ENV**: `NODE_ENV === 'production'`
  - Disables Prisma client caching to prevent memory leaks

### Environment Variables
```
DATABASE_URL="file:./dev.db"              # Dev: local, Prod: overridden by TURSO_*
TURSO_DATABASE_URL=""                     # Production Turso URL
TURSO_AUTH_TOKEN=""                       # Production Turso auth token
AUTH_SECRET="icp_change_this_in_production_now"
AUTH_USERNAME="admin"
AUTH_PASSWORD="admin"
```
