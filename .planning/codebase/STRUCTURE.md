## 2. Key File Purposes

### lib/calc.ts
Pure functions, no DB dependencies. Testable in isolation.
- `calcularNota(tipo, polaridade, valorRealizado, metaAlvo, metaMinima, metaMaxima)` → 0–120
- `calcularPremio(salarioBase, targetMultiploSalarial, nota, pesoNaCesta)` → R$
- `normalizarLinhaColaborador(raw)` → strips BOM, trims whitespace
- `parseSalario(raw)` → "8.000,50" → 8000.5

### lib/validators.ts
Row-level validation for bulk imports. Returns error string or validated data.
- `validarLinhaColaborador(raw)` — matricula, email, salary, admission date
- `validarLinhaMeta(raw)` — indicadorCodigo, peso ∈ [0,100], alvo > 0, min < alvo < max
- `validarLinhaRealizacao(raw)` — matricula, meta code, valorRealizado

### lib/janelas.ts
- `isJanelaAberta(cicloId, mes, ano)` → `{ aberta, janela }` — used in realizacoes POST

### app/page.tsx (~2200 lines)
Monolithic main dashboard. Contains:
- 15+ TypeScript interfaces
- 5 tab panels: dashboard, metas, agrupamentos, colaboradores, cadastros
- All CRUD handlers (handleCriarMeta, handleReset, handleColabImport, etc.)
- Template download functions (handleDownloadTemplateMetaas, etc.)

### components/MasterDashboard.tsx
- Fetches `/api/dashboard?cicloId=`
- Renders: JanelaApuracao status, KPI cards (4-col), bonus pool chart, top 5 performers, alerts

### components/PainelGestor.tsx
- Fetches scorecard for each subordinate (Promise.all)
- Renders: team summary, ranked table, monthly evolution chart

### components/CockpitColaborador.tsx
- Fetches `/api/scorecard?colaboradorId=&cicloId=`
- Renders: radial gauge (notaYTD), KPI cards, meta table, bottom 3 ("ofensores"), bar chart
