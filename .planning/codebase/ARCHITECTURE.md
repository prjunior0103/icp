## 1. System Overview

**ICP (Incentivo de Curto Prazo)** is a Brazilian performance management and short-term bonus calculation system:

- **Bonus/Performance Management**: Tracks employee goals, records achievements, calculates performance scores (nota: 0–120), and projects bonuses
- **Fiscal Cycle Management**: Organizes goals within fiscal years (SETUP → ATIVO → ENCERRADO phases)
- **Hierarchical Meta Cascading**: Corporate goals cascade to departments and individuals with weight distribution
- **Approval Workflows**: Multi-stage approval for metas, realizações, and HR movements
- **Role-Based Access**: Four tiers — Guardião, BP, Gestor, Colaborador
- **Dynamic Reporting**: Dashboards for executives (MasterDashboard), managers (PainelGestor), and employees (CockpitColaborador)

## 3. Business Logic: Nota & Prêmio Calculation

### Nota (`calcularNota()` — lib/calc.ts)

```
PROJETO_MARCO:
  nota = valorRealizado >= 1 ? 100 : 0

MENOR_MELHOR (e.g. cost, defects):
  nota = (metaAlvo / valorRealizado) * 100
  if valorRealizado > metaMaxima → nota = 0
  cap at 120

MAIOR_MELHOR (e.g. revenue):
  nota = (valorRealizado / metaAlvo) * 100
  if valorRealizado < metaMinima → nota = 0
  cap at 120
```

### Prêmio (`calcularPremio()` — lib/calc.ts)

```
prêmio = salarioBase × targetMultiploSalarial × (nota / 100) × (pesoNaCesta / 100)
```

## 7. Key Patterns

- **Upsert** — MetaColaborador and Realizações use `prisma.*.upsert` to avoid duplicates
- **Promise.all** — Dashboard and Scorecard routes parallelize 8+ queries
- **Soft delete** — Metas use `status: CANCELADO`, Colaboradores use `ativo: false`
- **Recursive hierarchy** — `getSubordinadosRecursivo()` in agrupamentos traverses full manager tree (BFS)
- **Janela validation** — Every realizacao POST checks: ciclo ATIVO + janela open or waiver approved
- **Audit trail** — MetaHistorico created on every meta field change
- **Brazilian number parsing** — `parseSalario()`: "8.000,50" → 8000.5
